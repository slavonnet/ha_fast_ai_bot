## Постановка
Контрактные snapshot тесты

## TDD UserStory (5)
1. As a platform maintainer, I want 'Контрактные snapshot тесты' реализован(а), so that этап 2 можно закрыть без функциональных дыр.
2. As a QA engineer, I want тесты happy-path для 'Контрактные snapshot тесты', so that базовый сценарий стабильно проходит.
3. As a QA engineer, I want edge-case тесты для 'Контрактные snapshot тесты', so that система корректно ведет себя на границах входных данных.
4. As an operator, I want error/fallback сценарии для 'Контрактные snapshot тесты', so that отказ не приводит к ложному успеху.
5. As a release owner, I want docs обновлены по 'Контрактные snapshot тесты', so that изменения воспроизводимы без чтения PLAN.ml.

## Acceptance criteria
- [ ] Реализован функционал подзадачи
- [ ] Есть failing->green цикл тестов
- [ ] Негативные/пограничные сценарии покрыты тестами
- [ ] Обновлена документация по изменению
- [ ] Нет незафиксированного техдолга
