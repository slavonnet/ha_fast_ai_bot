# RULES.CODE_DEVELOPMENT

- Разработка только после шагов полноты задачи, TDD дизайна и архитектуры.
- Запрещено оставлять заглушки без явной issue и пометки как техдолг.
- Нельзя игнорировать часть постановки: несделанные части переводятся в дочерние issue.
- Код должен быть трассируем к пунктам TDD UserStory.
