# RULES.TEST_DEVELOPMENT

- Для каждой подзадачи 5-10 UserStory тестов верхнего уровня.
- Для каждой темы постановки должны быть happy-path, edge-case и failure/fallback сценарии.
- TDD цикл обязателен: failing -> minimal implementation -> green -> refactor.
- Тесты должны покрывать реализацию с запасом по возможным вариантам.
